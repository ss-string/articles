# 핫뉴스 최신 정책 페이지 구현 계획

## 목표

Toss 핫뉴스 페이지가 `perspective_key` 기준 중복 정리 정책을 따르도록 바꾼다. 기본 목록은 `전체` 범위로 latest view 전체를 보여주고, 사용자가 `오늘`을 선택하면 오늘 발행 리포트만 보여준다. 상세 화면은 기존 리포트 본문 형식을 유지하면서 하단 디버그 영역에만 문서 상태와 원본 추적 정보를 표시한다.

## 구조

데이터 계층은 최신 목록 조회와 원 테이블 이력 조회를 분리한다. 모델 계층은 DB 추적 필드를 표시 가능한 형태로 정규화한다. 화면 계층은 일반 사용자에게 필요한 리포트 본문과 운영 확인용 디버그 정보를 분리한다.

## 사용 기술

React 19, TypeScript, Vite, Vitest, Testing Library, Supabase JS를 사용한다.

## 1단계: 데이터 모델과 조회 함수

### 수정 파일

- `src/hot-news/model.ts`
- `src/hot-news/model.test.ts`
- `src/hot-news/api.ts`
- `src/hot-news/api.test.ts`

### 작업 내용

- `HotNewsReport`에 다음 필드를 추가한다.
  - `displayTitle`
  - `perspectiveKey`
  - `runSlot`
  - `isLatest`
  - `changeStatus`
  - `changeReason`
  - `materialChangeScore`
  - `sourceNewsIds`
  - `companyCodes`
  - `positionMap`
  - `updatedAt`
  - `displayUpdatedAt`
- `stripIssueDatePrefix`로 `YYYY-MM-DD ` 제목 접두어를 제거한다.
- `formatKoreanDateTime`으로 Asia/Seoul 기준 `YYYY-MM-DD HH:mm` 표시값을 만든다.
- `getHotNewsChangeStatusLabel`로 상태를 한글 라벨로 바꾼다.
  - `initial`: 초기 문서
  - `refresh`: 새로고침 업데이트
  - `material_change`: 중요 변경 업데이트
  - `deduplicated`: 중복 정리됨
  - 알 수 없는 문자열: 원문 유지
  - 값 없음: 상태 없음
- `queryHotNewsReportRows(issueDate?)`는 `toss_wts_hot_news_latest_reports`를 조회한다.
- `queryHotNewsReportHistoryRows(issueDate, perspectiveKey)`는 `toss_wts_hot_news_reports`에서 같은 발행일과 관점 키의 이력을 조회한다.

### 검증

- `rtk npm test -- src/hot-news/model.test.ts src/hot-news/api.test.ts`
- `rtk npx tsc -b --pretty false`

## 2단계: 오늘과 전체 조회 훅

### 수정 파일

- `src/hot-news/useHotNewsReports.ts`
- `src/hot-news/useHotNewsReports.test.tsx`

### 작업 내용

- `getTodayInSeoul()`을 추가해 Asia/Seoul 기준 오늘 날짜를 계산한다.
- `useHotNewsReports`는 `scope` 값에 따라 데이터를 불러온다.
  1. `all`: `issue_date` 조건 없이 latest view를 조회한다.
  2. `today`: 오늘 날짜로 latest view를 조회한다.
  3. 오늘 결과가 없어도 다른 발행일로 다시 조회하지 않고 빈 목록을 정상 상태로 둔다.
- `useHotNewsReportHistory`를 추가한다.
  - `enabled`, `issueDate`, `perspectiveKey`가 모두 유효할 때만 이력을 조회한다.
  - 조건이 부족하면 조회하지 않고 빈 성공 상태를 반환한다.

## 3단계: 뉴스 페이지와 상세 모달

### 수정 파일

- `src/components/HotNewsReportsPage.tsx`
- `src/components/HotNewsReportModal.tsx`
- `src/components/HotNewsReportsPage.test.tsx`
- `src/styles.css`

### 작업 내용

- 페이지 주입 속성에 `queryRows`, `queryHistoryRows`, `today`를 추가한다.
- `오늘`과 `전체` 토글을 추가한다.
- 최신 발행일 안내나 오늘 업데이트 안내 문구는 표시하지 않는다.
- 카드에는 다음 정보만 표시한다.
  - 업데이트 시각 또는 발행일
  - 날짜 접두어가 제거된 제목
  - 첫 번째 핵심 요약
- 카드에는 별도 관점 키, 실행 슬롯, 변경 상태 조합 문자열을 표시하지 않는다.
- 모달 제목은 `displayTitle`을 사용한다.
- 모달 상단에는 업데이트 시각을 표시한다.
- 기존 상세 섹션을 유지한다.
  - `TL;DR`
  - `시장 해석`
  - `기업별 근거`
  - `주요 기사`
- `bull`, `bear`, `neutral` 톤 클래스를 유지한다.
- 모달 하단 `디버그 상태` 섹션에만 다음 정보를 표시한다.
  - 문서 상태
  - 업데이트 시각
  - 이력 조회 상태
  - 중요 변경 이력
  - 정리된 중복 요약
  - 원본 추적 필드
- 원본 추적 필드는 접이식으로 표시한다.
  - 관점 키
  - 실행 슬롯
  - 뉴스 ID
  - 종목 코드
  - 포지션 맵
- 배경 클릭 영역은 포커스 가능한 버튼으로 노출하지 않고, 모달 안의 닫기 버튼을 기본 접근 경로로 둔다.

- `rtk npx tsc -b --pretty false`

### 브라우저 확인

`/articles/finance/hot-news`에서 다음을 확인한다.

- 기본 진입 시 `전체` 토글이 선택되어 있다.
- `오늘` 토글을 선택하면 오늘 발행일 리포트만 보인다.
- 오늘 데이터가 없어도 최신 발행일 안내가 보이지 않는다.
- 카드 제목과 상세 제목에 날짜 접두어가 없다.
- 업데이트 시각이 보인다.
- 기존 리포트 본문 구조가 유지된다.
- 기업별 근거의 방향성 톤이 유지된다.
- 문서 상태와 원본 추적 정보는 하단 디버그 영역에만 보인다.
- 내부 추적 조합 문자열은 본문 상단에 보이지 않는다.
