# 컨센서스 상세 증권사 보조 링크 구현 계획

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 컨센서스 상세 모달의 기존 메시지와 구조를 유지하면서, `ai_consensus_summary_reports.analysis.excludedReports`의 `supportingUrls`를 기준으로 증권사별 목표가 행에 보조 링크를 연결한다.

**Architecture:** `sourceReports`는 읽거나 렌더링하지 않는다. Supabase `analysis` JSON에서 `excludedReports`만 정규화하고, 상세 모달에서 `securitiesFirms[].name`과 `excludedReports[].firm`을 정확히 매칭해 첫 번째 `supportingUrls[0]`를 증권사 행 링크로 사용한다. AI 컨센서스 요약, 주요 리스크, 목표주가 범위, 증권사별 목표가의 기존 문구와 위치는 유지하고, 링크가 있는 행에만 작은 `보조 링크` 배지를 추가한다.

**Tech Stack:** React 19, TypeScript, Vite, Vitest, Testing Library, Supabase REST JSON row.

---

## 고정 설계

- 기존 상세 모달의 핵심 메시지는 유지한다.
  - `AI 컨센서스 요약` 문구 유지
  - `주요 리스크` 문구 유지
  - `증권사별 목표가` 문구 유지
  - 모달 헤더의 `컨센서스 리포트 n개 기반 · AI 리포트 업데이트 ...` 문장 유지
- `sourceReports`는 사용하지 않는다.
  - 타입에 추가하지 않는다.
  - 정규화하지 않는다.
  - 렌더링하지 않는다.
  - 공식 출처 건수 문구를 헤더에 추가하지 않는다.
- `excludedReports`만 사용한다.
  - `excludedReports[].firm`이 `securitiesFirms[].name`과 같은 경우만 매칭한다.
  - 같은 증권사에 여러 항목이 있으면 `supportingUrls`가 1개 이상인 첫 번째 항목을 사용한다.
  - `supportingUrls[0]`만 대표 링크로 사용한다.
  - 링크는 보조 링크다. 공식 검증 링크라고 표현하지 않는다.
- 별도 `출처 미확인 analysis 항목` 섹션은 만들지 않는다.
- 링크가 있는 증권사 행은 새 탭으로 열린다.
  - `target="_blank"`
  - `rel="noreferrer"`
- 링크가 없는 증권사 행은 현재와 같은 비클릭 행으로 둔다.
- LG전자 실제 DB row에서 확인한 표시 의도:
  - `삼성증권`, `교보증권`, `미래에셋증권`, `신한투자증권` 등은 `excludedReports.supportingUrls`에 링크 후보가 있어 `보조 링크` 배지를 표시할 수 있다.
  - 이 링크들은 `excludedReports`에 있으므로 “공식 출처”가 아니라 “보조 링크”로만 표시한다.

function parseExcludedReports(value: unknown): ConsensusExcludedReport[] {
  if (!Array.isArray(value)) {
    return [];
  }

  return value.flatMap((item) => {
    const record = parseRecord(item);
    const title = parseText(record?.title);
    const firm = parseText(record?.firm);
    const reason = parseText(record?.reason);

    if (!record || !title || !firm || !reason) {
      return [];
    }

    return [
      {
        bulletNo: parseText(record.bulletNo),
        title,
        firm,
        publishedDate: parseText(record.publishedDate),
        supportingUrls: parseTextArray(record.supportingUrls),
        reason,
      },
    ];
  });
}

export function findFirmSupportingLink(
  report: ConsensusSummaryReport | null,
  firmName: string,
): ConsensusFirmSupportingLink | null {
  const matchedReport = report?.excludedReports.find(
    (excludedReport) => excludedReport.firm === firmName && excludedReport.supportingUrls.length > 0,
  );

  if (!matchedReport) {
    return null;
  }

  return {
    url: matchedReport.supportingUrls[0],
    title: matchedReport.title,
    firm: matchedReport.firm,
    publishedDate: matchedReport.publishedDate,
    bulletNo: matchedReport.bulletNo,
    reason: matchedReport.reason,
  };
}
