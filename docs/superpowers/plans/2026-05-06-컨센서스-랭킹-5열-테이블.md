# 컨센서스 랭킹 5열 테이블 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 컨센서스 랭킹 테이블을 `종목 | 현재가 | 적정가 | 갭 | 적정가 대비 현재가` 5열 구조로 바꾸고, 마지막 컬럼의 지난 1개월 컨센서스 증가율 배지를 제거한다.

**Architecture:** 변경은 `ConsensusTable`의 렌더링과 관련 CSS, 테스트에 한정한다. 기존 데이터 모델과 정렬, 상세 팝업, 현재가 비율 계산 방식은 유지한다.

**Tech Stack:** React 19, TypeScript, Vitest, Testing Library, CSS

- [ ] **Step 1: Write failing tests**

In `src/components/ConsensusTable.test.tsx`, update the table semantics test to expect `적정가` and `적정가 대비 현재가`, and to stop expecting `적정주가` / `지난 1개월 대비 컨센서스 증가`.

Add this test inside `describe('ConsensusTable', () => { ... })`:

```tsx
it('renders a five-column layout without the one month consensus change badge', () => {
  render(<ConsensusTable rows={[makeRow()]} onSelect={() => undefined} />);

  const table = screen.getByRole('table', { name: '컨센서스 랭킹 테이블' });

  expect(within(table).getByRole('columnheader', { name: '종목' })).toBeInTheDocument();
  expect(within(table).getByRole('columnheader', { name: '현재가' })).toBeInTheDocument();
  expect(within(table).getByRole('columnheader', { name: '적정가' })).toBeInTheDocument();
  expect(within(table).getByRole('columnheader', { name: '갭' })).toBeInTheDocument();
  expect(within(table).getByRole('columnheader', { name: '적정가 대비 현재가' })).toBeInTheDocument();
  expect(within(table).queryByRole('columnheader', { name: '적정주가' })).not.toBeInTheDocument();
  expect(within(table).queryByRole('columnheader', { name: '지난 1개월 대비 컨센서스 증가' })).not.toBeInTheDocument();
  expect(screen.queryByText('▲ +6.8%', { selector: '.consensus-badge' })).not.toBeInTheDocument();
});
```

Remove or rewrite the existing tests that require `.consensus-badge` to render for missing, negative, and zero one-month consensus changes. Those assertions should no longer exist because the badge is intentionally removed from the table.
