# 분석자료실 피드백 개선 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 확정된 피드백 설계에 따라 분석자료실 브랜딩, 사이드바 닫힘 레이아웃, 핫뉴스 tone 컬러, 컨센서스 중복 제목, 매크로 판단 컬러를 개선한다.

**Architecture:** 기존 라우팅 구조와 데이터 모델은 유지한다. 의미 판정은 컴포넌트 내부의 작은 tone helper로 className을 생성하고, 공통 시각 표현은 `src/styles.css`에 집중한다. CSS Grid 자동 배치 버그는 `workspace` 컬럼을 명시하고 본문 컨테이너에 중앙 정렬을 적용해 해결한다.

**Tech Stack:** React 19, TypeScript, Vite, Vitest, Testing Library, CSS.

function getEvidenceToneClass(position: string | null) {

    case 'neutral':

Add this helper near the top of `src/components/MacroRegimePage.tsx`:

```tsx
function getMacroJudgmentToneClass(judgment: string) {
  const normalized = judgment.trim().toLowerCase();

  if (['elevated', 'contracting', 'tight', 'restrictive', 'scarce'].includes(normalized)) {
    return 'tone-bear';
  }

  if (normalized === 'neutral' || normalized === '-') {
    return 'tone-neutral';
  }

  return 'tone-bull';
}
```
