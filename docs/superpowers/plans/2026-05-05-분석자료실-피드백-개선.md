# 분석자료실 피드백 개선 구현 계획

> **에이전트 작업자 안내:** 이 계획은 `superpowers:subagent-driven-development`(권장) 또는 `superpowers:executing-plans`를 사용해 작업별로 구현한다. 진행 상황 추적에는 체크박스(`- [ ]`) 문법을 사용한다.

**목표:** 확정된 피드백 설계에 따라 분석자료실 브랜딩, 사이드바 닫힘 레이아웃, 핫뉴스 톤 컬러, 컨센서스 중복 제목, 매크로 판단 컬러를 개선한다.

**구조:** 기존 라우팅 구조와 데이터 모델은 유지한다. 의미 판정은 컴포넌트 내부의 작은 톤 헬퍼로 `className`을 생성하고, 공통 시각 표현은 `src/styles.css`에 집중한다. CSS Grid 자동 배치 버그는 `workspace` 컬럼을 명시하고 본문 컨테이너에 중앙 정렬을 적용해 해결한다.

**기술 스택:** React 19, TypeScript, Vite, Vitest, Testing Library, CSS.

```tsx
function getMacroJudgmentToneClass(judgment: string) {
  const normalized = judgment.trim().toLowerCase();

  if (['elevated', 'contracting', 'tight', 'restrictive', 'scarce'].includes(normalized)) {
    return 'tone-bear';
  }

  if (normalized === 'neutral' || normalized === '-') {
    return 'tone-neutral';
  }

  return 'tone-bull';
}
```

카드의 축약 판단 표시를 수정한다.
