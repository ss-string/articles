# 핫뉴스 최신 정책 페이지 설계

## 배경

Toss 핫뉴스 리포트 DB는 `perspective_key` 기준으로 중복 관점을 정리한다. 기본 뉴스 페이지는 원 테이블의 모든 이력 행이 아니라 `public.toss_wts_hot_news_latest_reports` 뷰의 최신 행만 보여줘야 한다. 상세 화면에서는 같은 `issue_date + perspective_key`에 묶인 원 테이블 이력을 확인할 수 있어야 하지만, 이 정보는 일반 독자에게 먼저 보이는 본문을 방해하면 안 된다.

## 목표

- 핫뉴스 기본 목록은 `전체` 범위로 latest view 전체를 조회한다.
- 사용자가 `오늘`을 선택하면 `issue_date = 오늘` 조건으로만 조회한다.
- 오늘 데이터가 없어도 다른 발행일을 오늘 행처럼 보정하지 않는다.
- 카드와 상세 제목은 `2026-05-07 AI 반도체 인프라`가 아니라 `AI 반도체 인프라`처럼 날짜 접두어를 제거해 표시한다.
- 기존 상세 구조인 `TL;DR`, `시장 해석`, `기업별 근거`, `주요 기사`를 유지한다.
- 기업별 근거의 `bull`, `bear`, `neutral` 톤 표시를 유지한다.
- 업데이트 시각은 `2026-05-07 21:14` 형식으로 노출한다.
- `2026-05-07 · ai_infra · initial` 같은 내부 추적 문자열은 일반 본문에 노출하지 않는다.
- 문서가 초기 문서인지, 새로고침인지, 중요 변경인지, 중복 정리인지 여부는 상세 하단의 디버그 영역에만 표시한다.

## 데이터 조회 정책

기본 목록은 `toss_wts_hot_news_latest_reports`를 조회한다. 조회 범위는 다음과 같다.

- `전체`: `issue_date` 조건 없이 latest view를 조회한다.
- `오늘`: Asia/Seoul 기준 오늘 날짜를 `YYYY-MM-DD`로 계산하고 `issue_date = 오늘` 조건으로 latest view를 조회한다.
- `오늘` 결과가 비어 있으면 빈 상태를 표시하고, 최신 발행일 안내나 오늘 업데이트 안내 문구는 표시하지 않는다.

상세 이력은 사용자가 카드를 선택했을 때 원 테이블 `toss_wts_hot_news_reports`를 조회한다. 필터는 `issue_date`와 `perspective_key`를 사용하고, 정렬은 `is_latest desc`, `run_slot desc`를 사용한다.

## 화면 구조

목록 영역은 기존 카드 UI를 유지하되 `오늘`과 `전체` 토글을 추가한다. 카드에는 업데이트 시각, 날짜 접두어가 제거된 관점 제목, 첫 번째 핵심 요약만 표시한다. `change_status` 값은 카드에 노출하지 않고 상세 하단 디버그 영역에서만 표시한다.

상세 모달은 기존 리포트 형식을 유지한다.

- 상단: 제목, 업데이트 시각
- 본문: `TL;DR`
- 본문: `시장 해석`
- 본문: `기업별 근거`
- 본문: `주요 기사`
- 하단: `디버그 상태`

디버그 상태에는 문서 상태 라벨, 업데이트 시각, 정리된 중복 행 요약, 필요 시 원본 추적 필드를 접이식으로 표시한다. `perspective_key`, `run_slot`, `source_news_ids`, `company_codes`, `position_map`은 디버그 영역 밖으로 노출하지 않는다.

## 상태 라벨

- `initial`: 초기 문서
- `refresh`: 새로고침 업데이트
- `material_change`: 중요 변경 업데이트
- `deduplicated`: 중복 정리됨
- 그 외 값: 원문 문자열을 하단 디버그에만 표시

## 오류 처리

기본 목록 조회가 실패하면 기존 에러 패널을 사용한다. `오늘` 범위에서 데이터가 없으면 빈 상태를 정상 표시한다. 상세 이력 조회가 실패하면 상세 본문은 유지하고, 하단 디버그 영역에 이력 조회 실패 메시지만 표시한다.

## 테스트 기준

- 기본 진입 시 `전체` 범위로 latest view 전체를 조회한다.
- `오늘` 토글을 선택하면 오늘 latest view 결과만 사용한다.
- 오늘 데이터가 없으면 빈 상태를 표시하고 최신 발행일 안내를 표시하지 않는다.
- 기본 목록 조회 대상은 `toss_wts_hot_news_latest_reports`다.
- 상세 이력 조회 대상은 `toss_wts_hot_news_reports`다.
- 제목에서 날짜 접두어가 제거된다.
- 업데이트 시각이 `YYYY-MM-DD HH:mm`으로 표시된다.
- `bull`, `bear`, `neutral` 톤 클래스가 기존처럼 유지된다.
- `perspective_key`와 `change_status` 조합 문자열은 본문 상단에 노출되지 않는다.
- 문서 상태는 상세 하단 디버그 영역에만 표시된다.
