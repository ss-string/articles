# 인공지능 투자 리포트 지면 설계

## 목표

`금융 > AI 분석 리포트`에서 `investment_recommendation_reports`의 투자 추천 리포트를 읽어 종목별 AI 분석 지면을 제공한다. 진입 시에는 상단 검색으로 종목을 찾고, 검색 결과에서 선택한 종목의 리포트 이력을 최신순에서 과거순으로 보여준다. 기본 선택은 전체 대표 종목 중 가장 최근 리포트를 가진 종목과 해당 종목의 최신 리포트다.

삼성전자 실제 row를 기준으로 확정한 핵심 요구사항은 다음과 같다.

- 상단 검색은 `stock_name`과 `stock_code`로 대표 종목을 찾는다.
- 검색 결과의 대표 종목은 `stock_code` 기준 최신 row만 사용한다.
- 좌측 목록은 선택된 종목의 모든 리포트를 `updated_at` 내림차순으로 보여준다.
- 기본 선택은 최신 리포트와 `totalScore` 종합 지면이다.
- `totalScore` 선택 시 별도 선택 상세를 만들지 않고 종합 리포트 지면을 유지한다.
- `momentum`, `technical`, `valuation` 선택 시 해당 agent 지면으로 전환한다.
- agent 지면은 `summary`, `findings`, `risks`, `limitations` 네 영역으로 구성한다.
- agent findings는 `bull` / `bear` 뱃지와 `confidence` 값, `evidence`를 모두 노출한다.
- agent findings의 `metric`은 화면에 노출하지 않는다.
- 종합 지면의 `content_md`는 DB 문자열을 기준으로 Markdown 문서 형식에 맞게 렌더링한다.
- 종합 지면의 `bullFindings`, `bearFindings`는 뱃지 없이 일반 목록으로 보여준다.
- `actionPlan` 박스 상단에는 `current_price`를 현재가로 표시한다.

## 데이터 소스

주 데이터는 `investment_recommendation_reports` 테이블이다. 확인된 주요 컬럼은 다음과 같다.

- `id`
- `market`
- `stock_code`
- `stock_name`
- `issue_date`
- `recommendation`
- `current_price`
- `total_score`
- `valuation_score`
- `momentum_score`
- `technical_score`
- `content_md`
- `report_payload`
- `agent_outputs`
- `source_payload`
- `created_at`
- `updated_at`

`report_payload`는 종합 지면의 구조화 정보를 제공한다.

- `contentMd`
- `actionPlan`
- `bullFindings`
- `bearFindings`
- `riskChecklist`
- `investmentThesis`
- `recommendation`
- `totalScore`
- `scores`
- `stanceSummary`

`agent_outputs`는 agent 상세 지면의 원천 데이터다.

- `agent_outputs.momentum`
- `agent_outputs.technical`
- `agent_outputs.valuation`

각 agent 객체는 다음 구조를 사용한다.

- `summary`
- `score`
- `stance`
- `findings[]`
- `risks[]`
- `dataQuality.limitations[]`

각 `findings[]` 항목은 다음 구조를 사용한다.

- `type`: `bull` 또는 `bear`
- `evidence`: 화면에 노출할 근거 문장
- `confidence`: `high`, `medium`, `low` 등 신뢰도 값
- `metric`: 내부 판단 기준. 화면에는 노출하지 않는다.

## 진입, 검색, 목록 정책

`AI 분석 리포트` 메뉴로 진입하면 상단에 종목 검색 영역을 보여준다. 검색 입력은 `stock_name`과 `stock_code`를 대상으로 필터링한다. 검색 결과는 대표 종목 버튼으로 표시하며, 대표 종목은 `stock_code` 기준 최신 row 하나만 사용한다.

대표 종목 생성 규칙은 다음과 같다.

1. 모든 row를 `stock_code`로 그룹화한다.
2. 각 그룹에서 최신 row를 대표 리포트로 선택한다.
3. 최신성은 `updated_at`을 우선으로 판단한다.
4. `updated_at`이 없으면 `created_at`을 보조 기준으로 사용한다.
5. 검색 결과 기본 정렬은 대표 리포트의 `updated_at` 내림차순이다.

검색 결과 대표 버튼에는 다음 정보를 표시한다.

- `stock_code`
- `stock_name`
- 해당 종목의 리포트 수

검색어가 비어 있으면 대표 종목 전체를 최신순으로 표시할 수 있다. 검색어가 있으면 종목명 또는 종목코드에 검색어가 포함된 대표 종목만 표시한다.

좌측 목록은 전체 대표 종목 목록이 아니다. 좌측 목록에는 현재 선택된 종목의 리포트 이력만 표시한다. 동일 종목의 모든 리포트를 `updated_at` 기준 최신순에서 과거순으로 보여주며, 가장 최신 리포트에는 `최신 리포트`, 나머지에는 `과거 리포트` 표기를 붙인다.

## 종목 상세 이력

검색 결과에서 종목 대표 버튼을 선택하면 해당 `stock_code`의 상세 지면으로 진입한다. 상세 지면의 좌측에는 같은 기업의 리포트 이력을 최신순으로 모두 보여준다.

리포트 이력 항목은 다음 정보를 표시한다.

- `updated_at`
- `recommendation`
- `total_score`
- `momentum_score`
- `technical_score`
- `valuation_score`

선택된 종목의 가장 최신 리포트를 기본 선택한다. 사용자가 이전 리포트를 선택하면 다음 영역이 해당 row 기준으로 함께 갱신된다.

- 헤더 메타 정보
- 상세 상단 투자 가설 요약
- 추천 의견
- 현재가
- 점수 카드
- `content_md`
- `actionPlan`
- `bullFindings`
- `bearFindings`
- `riskChecklist`
- agent 상세 지면

이전 리포트에 `agent_outputs`가 있으면 최신 리포트와 같은 agent 상세 지면을 사용한다. 이전 리포트에 `agent_outputs`가 없거나 일부 agent가 비어 있으면 `report_payload` 기반 요약 fallback을 보여주되, 상세 `evidence`와 `confidence`가 없음을 명시한다.

## 종합 지면

기본 지면은 `totalScore` 종합 지면이다. `totalScore` 점수 카드는 기본 선택 상태로 표시한다.

종합 지면은 다음 영역으로 구성한다.

1. `content_md`
2. `actionPlan`
3. `bullFindings`
4. `bearFindings`
5. `riskChecklist`
6. DB row 매핑 또는 디버그 메타

`content_md`는 DB 문자열을 원문 기준으로 사용한다. 문장을 임의로 요약하거나 재작성하지 않는다. 화면 렌더링은 Markdown 형식을 적용한다.

- `##`는 소제목으로 렌더링한다.
- `###`는 하위 소제목으로 렌더링한다.
- 빈 줄로 나뉜 문단은 별도 문단으로 렌더링한다.
- `**텍스트**`는 굵게 렌더링한다.
- 지원하지 않는 Markdown 구문은 원문 텍스트가 손상되지 않게 일반 텍스트로 남긴다.

`actionPlan` 박스 상단에는 `current_price`를 표시한다. 값이 있으면 원화 형식으로 표시한다.

예시는 다음과 같다.

- 라벨: `현재가`
- 값: `273,500원`

`current_price`가 없으면 `현재가 없음`을 표시한다.

`actionPlan` 본문은 `hold`, `entry`, `exitOrReview` 순서로 표시한다. 이 중 없는 항목은 생략한다.

종합 지면의 `bullFindings`, `bearFindings`는 각 섹션 제목으로 구분하되, 개별 항목에는 `bull` / `bear` 뱃지를 붙이지 않는다. 뱃지는 agent findings 전용 표현이다.

## Agent 지면

`momentum`, `technical`, `valuation` 점수 카드를 선택하면 종합 지면 대신 선택한 agent 지면을 보여준다.

각 agent 지면은 네 영역으로 구성한다.

1. `summary`
2. `findings`
3. `risks`
4. `limitations`

상단에는 agent 이름, score, stance를 표시한다.

예시는 다음과 같다.

- `Momentum Agent`
- `score 78`
- `stance bull`

`summary`는 `agent_outputs.{agent}.summary`를 그대로 표시한다.

`findings`는 `agent_outputs.{agent}.findings` 전체를 누락 없이 렌더링한다. 예를 들어 삼성전자 최신 row의 `momentum.findings`는 6개이며, 6개를 모두 보여줘야 한다.

각 finding 카드는 다음 요소로 구성한다.

- `type` 뱃지: `bull` 또는 `bear`
- `confidence` 뱃지: `high`, `medium`, `low` 같은 값만 표시한다. `confidence: high`처럼 라벨을 반복하지 않는다.
- `evidence`: 근거 문장 본문

`metric`은 화면에 표시하지 않는다. `metric`은 내부 판단 기준으로만 사용한다.

`risks`는 `agent_outputs.{agent}.risks` 전체를 누락 없이 목록으로 표시한다.

`limitations`는 `agent_outputs.{agent}.dataQuality.limitations` 전체를 누락 없이 목록으로 표시한다.

## 삼성전자 기준 확정 데이터 예시

최신 삼성전자 row에서 확인한 값은 다음과 같다.

- `stock_code`: `005930`
- `stock_name`: `삼성전자`
- `recommendation`: `매수`
- `current_price`: `273500`
- `issue_date`: `2026-05-17`
- `updated_at`: `2026-05-17T09:26:10.444135+00:00`
- `total_score`: `69`
- `momentum_score`: `78`
- `technical_score`: `74`
- `valuation_score`: `56`

삼성전자 최신 row의 `agent_outputs` 항목 수는 다음과 같다.

- `momentum.findings`: 6개
- `momentum.risks`: 5개
- `momentum.dataQuality.limitations`: 5개
- `technical.findings`: 2개
- `technical.risks`: 5개
- `technical.dataQuality.limitations`: 3개
- `valuation.findings`: 5개
- `valuation.risks`: 4개
- `valuation.dataQuality.limitations`: 4개

이 숫자는 렌더링 누락을 방지하기 위한 검증 기준으로 사용한다.

## UI 구조

전체 레이아웃은 기존 분석자료실 디자인을 따른다.

- 상단: 종목명 또는 종목코드 검색
- 좌측: 선택된 종목의 리포트 이력 목록
- 우측 상단: 선택 리포트 분석 결과, 투자 가설 요약, 추천 의견, totalScore
- 우측 중단: `totalScore`, `momentum`, `technical`, `valuation` 점수 카드
- 우측 하단: 선택한 점수에 따른 지면

점수 카드 동작은 다음과 같다.

- `totalScore`: 종합 지면 유지
- `momentum`: Momentum Agent 지면으로 전환
- `technical`: Technical Agent 지면으로 전환
- `valuation`: Valuation Agent 지면으로 전환

추천 의견 박스는 다음처럼 단순하게 표시한다.

```text
매수
totalScore 69
```

`DB recommendation` 같은 라벨은 점수 박스 안에 노출하지 않는다.

## 상태 처리

로딩 상태는 `AI 분석 리포트를 불러오는 중입니다.`를 표시한다.

오류 상태는 Supabase 오류 메시지를 사용자에게 보여주되, 환경변수나 키 값은 노출하지 않는다.

데이터가 없으면 `표시할 AI 분석 리포트가 없습니다.`를 표시한다.

특정 row의 일부 필드가 비어 있으면 다음 fallback을 사용한다.

- `current_price` 없음: `현재가 없음`
- `content_md` 없음: `본문이 없습니다.`
- `actionPlan` 없음: actionPlan 박스 본문 생략
- `agent_outputs` 없음: agent 상세 대신 `agent_outputs가 없습니다.` 표시
- `findings` 없음: `표시할 발견사항이 없습니다.` 표시
- `risks` 없음: `표시할 리스크가 없습니다.` 표시
- `limitations` 없음: `표시할 한계가 없습니다.` 표시

## 테스트 기준

모델 정규화 테스트는 다음을 검증한다.

- raw row에서 `stock_code`, `stock_name`, `recommendation`, `current_price`, 점수, 날짜를 읽는다.
- `report_payload`의 `contentMd`, `actionPlan`, `bullFindings`, `bearFindings`, `riskChecklist`를 정규화한다.
- `agent_outputs`의 `summary`, `findings`, `risks`, `dataQuality.limitations`를 정규화한다.
- agent finding의 `type`, `evidence`, `confidence`는 보존한다.
- agent finding의 `metric`은 모델에 저장할 수 있지만 기본 렌더링에는 사용하지 않는다.
- 동일 `stock_code` 대표 종목은 최신 row만 남긴다.
- 종목 상세 이력은 같은 `stock_code`의 모든 row를 `updated_at` 내림차순으로 정렬한다.

컴포넌트 테스트는 다음을 검증한다.

- 상단 검색에서 종목명 또는 종목코드로 대표 종목을 찾을 수 있다.
- 좌측에는 선택된 종목의 리포트 이력만 최신순에서 과거순으로 표시된다.
- 기본 화면은 `totalScore` 종합 지면이다.
- 종합 지면의 `content_md`는 Markdown 제목과 문단으로 렌더링된다.
- 종합 지면의 findings에는 `bull` / `bear` 뱃지가 없다.
- actionPlan 박스 상단에 현재가가 표시된다.
- `momentum`을 선택하면 momentum agent 지면이 열린다.
- momentum findings 6개가 모두 표시된다.
- agent finding에는 `bull` / `bear` 뱃지와 `high` 같은 confidence 값이 표시된다.
- agent finding에는 `metric` 텍스트가 표시되지 않는다.
- agent risks와 limitations가 누락 없이 표시된다.
- 이전 리포트를 선택하면 본문과 점수, actionPlan, findings가 선택 row 기준으로 갱신된다.

## 제외 범위

이번 설계는 지면과 데이터 표시 정책을 확정한다. 다음 항목은 구현 범위에서 제외하거나 별도 설계로 다룬다.

- 실시간 가격 스트리밍
- 투자 주문 또는 매매 연동
- 외부 뉴스 원문 상세 뷰어
- agent 재실행 기능
- 리포트 삭제 또는 수정 기능
- 사용자별 관심 종목 저장

## 승인된 시안 기준

브라우저 목업 v8의 최종 방향을 기준으로 한다.

- 종합 지면은 `content_md` Markdown 렌더링과 actionPlan/current price 중심이다.
- agent 지면은 `summary`, `findings`, `risks`, `limitations` 네 영역이다.
- agent findings는 `type`, `confidence`, `evidence` 중심으로 보여준다.
- metric은 숨긴다.
- 종목 상세에서는 이전 리포트 선택이 가능하다.
