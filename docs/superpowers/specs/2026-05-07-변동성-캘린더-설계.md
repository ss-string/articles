# 변동성 캘린더 설계

## 목적

`public.stock_volatility_check_calendars` 테이블을 기반으로 45일 안의 한국/미국 주식시장 변동성 이벤트를 금융 탭의 새 지면에서 보여준다. 사용자는 이벤트일 기준 캘린더에서 매수관점에서 중요하게 보아야하는 시점을 빠르게 확인하고, 이벤트 카드를 선택해 해당 이벤트가 무엇인지와 왜 중요한지 짧게 확인한다.

## 확정 화면 방향

금융 탭에 `변동성 캘린더` 새 페이지를 추가한다. 경로는 `/finance/volatility-calendar`로 둔다.

화면은 기존 분석자료실 디자인 구조를 따른다.

- 기존 `AppShell`과 금융 하위 탭을 사용한다.
- 지면은 흰색 `dashboard-section` 카드 안에 구성한다.
- 반경은 기존처럼 8px를 사용한다.
- 청록색을 주요 강조색으로 사용하고, 위험 또는 Bear 성격은 빨강 계열 보조색을 사용한다.
- 별도 랜딩 화면이나 새로운 디자인 시스템은 만들지 않는다.

## 데이터 구조

조회 대상은 `public.stock_volatility_check_calendars`이다.

현재 확인한 top-level 컬럼은 다음과 같다.

- `id`
- `issue_date`
- `events`
- `created_at`
- `updated_at`

캘린더 표시는 `issue_date`가 아니라 `events[].date_kst` 이벤트일 기준으로 배치한다. `issue_date`는 표시하지 않는다. 상단 메타에는 `updated_at`을 `최신 업데이트 YYYY.MM.DD HH:mm` 형식으로 표시한다.

현재 확인한 `events` 내부 필드는 다음과 같다.

- `event`: 이벤트명
- `market`: `KR_STOCK`, `US_STOCK`
- `category`: 이벤트 분류
- `date_kst`: 이벤트일
- `time_kst`: 이벤트 시간
- `risk_level`: 0.0부터 1.0까지의 중요도 수치
- `buy_caution`: 매수 관점 주의 문구
- `check_points`: 체크포인트 문자열 배열

시장에서 사용하는 라벨은 다음처럼 변환한다.

- `KR_STOCK` → `한국`
- `US_STOCK` → `미국`

## 추가 데이터 계약

확정 시안에는 각 이벤트의 `Bull`/`Bear` 관점과 팝업의 짧은 설명이 포함된다. 현재 테이블 샘플에는 이 값이 별도 필드로 없으므로, 구현은 다음 우선순위를 따른다.

이벤트 관점은 `events[].stance`가 있으면 사용한다.

- 허용값: `bull`, `bear`
- 표시값: `Bull`, `Bear`

`stance`가 없으면 초기 구현에서는 카테고리 기반 파생 규칙을 사용한다. 물가, 중앙은행, 의사록, 휴장, 만기, 리밸런싱은 기본 `Bear`로 보고, 소비와 심리 관련 이벤트는 기본 `Bull`로 본다. 이 규칙은 데이터에 명시 필드가 들어오면 즉시 대체한다.

팝업 설명은 다음 필드가 있으면 우선 사용한다.

- `events[].event_brief`: 이 이벤트가 무엇인지
- `events[].importance_reason`: 왜 중요한지

필드가 없으면 `event`, `category`, `buy_caution`, `check_points`를 이용해 짧은 설명을 구성한다. 이때 장문의 생성형 설명은 하지 않고, 카테고리별 고정 문구와 기존 주의 문구만 조합한다.

## 조회 방식

최신 캘린더 row 1건을 조회한다.

```ts
.from('stock_volatility_check_calendars')
.select('id, issue_date, events, updated_at')
.order('issue_date', { ascending: false })
.limit(1)
```

조회한 row의 `events` 배열을 정규화한 뒤 `date_kst`, `time_kst`, `market`, `event` 순서로 정렬한다.

## 화면 구성

페이지 상단에는 기존 페이지 헤더가 표시된다.

- kicker: `매수 주의 캘린더`
- 제목: `변동성 캘린더`

본문 첫 섹션 제목은 `45일 이벤트 캘린더`로 둔다.

설명 문구는 다음 의미를 포함한다.

```text
이벤트일 기준으로 한국/미국 주요 변동성 이벤트를 배치합니다. 매수관점에서 중요하게 보아야하는 시점을 빠르게 확인하고, 각 이벤트가 Bull인지 Bear인지 함께 봅니다.
```

섹션 우측 메타에는 `최신 업데이트 YYYY.MM.DD HH:mm`을 표시한다.

시장 필터는 세 개만 제공한다.

- 전체
- 한국
- 미국

중요도 필터는 만들지 않는다.

## 캘린더 그리드

캘린더는 이벤트가 있는 날짜 박스를 그리드로 배치한다. 최상단의 `월`, `화`, `수` 같은 요일 헤더는 표시하지 않는다. 각 날짜 박스에 `YYYY.MM.DD 요일`을 직접 표시한다.

날짜 박스 안에는 해당 날짜의 이벤트 카드를 표시한다. 같은 날짜에 여러 이벤트가 있으면 카드가 세로로 쌓인다.

이벤트 카드에는 다음 정보를 표시한다.

- 시장 라벨
- 이벤트 시간
- 이벤트명
- `Bull` 또는 `Bear`
- `중요도 0.00`
- 중요도 progress bar

`risk_level`은 `중요도`로 표시한다. `강도`라는 단어는 사용하지 않는다. progress bar는 0.0부터 1.0까지의 값을 0%부터 100%까지로 환산한다.

날짜와 요일은 `date_kst` 기준으로 계산한다. 예를 들어 `2026-05-08`은 `2026.05.08 금`으로 표시한다.

## 이벤트 팝업

이벤트 카드를 선택하면 기존 앱의 모달 스타일을 따른 팝업을 연다.

팝업 상단에는 다음을 표시한다.

- `Event Detail`
- 이벤트명
- 이벤트일
- 시간
- 시장
- `Bull` 또는 `Bear`
- 닫기 버튼

팝업 본문은 다음 영역으로 나눈다.

- `무엇인지`: 이벤트의 성격을 한두 문장으로 설명한다.
- `왜 중요한지`: 가격 변동성 또는 매수 판단에 중요한 이유를 한두 문장으로 설명한다.
- `중요도`: 수치와 progress bar를 표시한다.
- `분류`: `category` 값을 표시한다.
- `체크포인트`: `check_points` 배열을 목록으로 표시한다.

팝업은 닫기 버튼, 배경 클릭, `Escape` 키로 닫을 수 있게 한다. 포커스는 팝업 내부에 유지한다.

## 상태 처리

다음 상태를 표시한다.

- 로딩: `변동성 캘린더를 불러오는 중입니다.`
- 오류: Supabase 오류 메시지를 비밀값 없이 표시한다.
- 빈 상태: 최신 row가 없거나 유효한 이벤트가 없으면 `표시할 변동성 이벤트가 없습니다.`
- 필터 빈 상태: 선택한 시장에 이벤트가 없으면 `선택한 시장에 표시할 이벤트가 없습니다.`

유효하지 않은 이벤트는 표시하지 않는다. 필수값은 `event`, `market`, `date_kst`, `risk_level`이다.

## 접근성

- 시장 필터는 버튼으로 구현하고 선택 상태를 `aria-pressed`로 표시한다.
- 이벤트 카드는 버튼으로 구현한다.
- progress bar는 수치 텍스트를 함께 제공해 색상만으로 의미를 전달하지 않는다.
- 팝업은 `role="dialog"`와 `aria-modal="true"`를 사용한다.
- 팝업 제목은 `aria-labelledby`로 연결한다.
- `Escape` 키와 닫기 버튼을 지원한다.

## 테스트

다음 항목을 테스트한다.

- Supabase row를 변동성 캘린더 모델로 정규화한다.
- `events[].date_kst` 기준으로 날짜와 요일을 표시한다.
- `updated_at`을 최신 업데이트 메타로 표시한다.
- `issue_date`를 화면의 기준일로 표시하지 않는다.
- 시장 필터는 전체, 한국, 미국만 표시한다.
- `KR_STOCK`, `US_STOCK`을 한국, 미국으로 변환한다.
- `risk_level`을 `중요도`와 progress bar로 표시한다.
- 카드에는 `강도` 텍스트가 표시되지 않는다.
- `Bull`/`Bear`가 표시되고 한글 관점 설명은 카드에 표시되지 않는다.
- 최상단 요일 헤더는 표시하지 않는다.
- 이벤트 카드 선택 시 팝업이 열린다.
- 팝업에 `무엇인지`, `왜 중요한지`, `중요도`, `분류`, `체크포인트`가 표시된다.
- 로딩, 오류, 빈 상태, 필터 빈 상태가 표시된다.
- 전체 테스트와 프로덕션 빌드가 통과한다.

## 수용 기준

- 금융 탭에 `변동성 캘린더` 페이지가 추가된다.
- 최신 `stock_volatility_check_calendars` row 1건을 조회한다.
- 이벤트는 `date_kst` 기준으로 배치한다.
- 상단에는 `updated_at` 기반 최신 업데이트 시간이 표시된다.
- 시장 필터는 전체, 한국, 미국만 제공한다.
- 이벤트 카드에는 시간, 이벤트명, 시장, `Bull`/`Bear`, `중요도 0.00`, progress bar가 표시된다.
- 카드에는 `매수 관점`, `매도 관점`, `강도` 텍스트가 표시되지 않는다.
- 캘린더 최상단에는 요일 헤더가 표시되지 않는다.
- 이벤트 카드 선택 시 기존 모달 톤의 팝업이 열리고, 이벤트가 무엇인지와 왜 중요한지를 간략히 보여준다.
