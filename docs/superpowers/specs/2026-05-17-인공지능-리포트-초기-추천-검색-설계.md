# 인공지능 리포트 초기 추천 검색 설계

## 목표

`금융 > AI 분석 리포트`의 최초 진입 경험을 바꾼다. 현재 지면은 데이터 로드 후 가장 최근 대표 종목을 자동 선택하고, 검색 입력에도 해당 종목명이 기본값으로 채워진다. 개선 후에는 검색 입력을 비워 두고, 아무 종목도 자동 선택하지 않는다.

검색어가 없는 초기 상태에서는 종목별 최신 리포트를 기준으로 `total_score`가 높은 10개 종목을 검색 결과 칩으로 먼저 보여준다. 사용자가 칩을 클릭하거나 검색어를 입력해 결과를 선택한 뒤에만 이력과 상세 본문을 렌더링한다.

## 확정 요구사항

- 최초 진입 시 검색 입력값은 빈 문자열이다.
- 최초 진입 시 선택된 종목은 없다.
- 검색어가 비어 있으면 선택 상태와 무관하게 검색 결과 영역에는 종목별 최신 리포트 기준 `total_score` 상위 10개 종목을 보여준다.
- 대표 종목은 기존 정책처럼 `stock_code`별 최신 리포트 1개를 사용한다.
- 상위 10개 정렬은 `total_score` 내림차순을 우선한다.
- `total_score`가 같거나 없는 경우에는 기존 최신성 정렬을 보조 기준으로 사용한다.
- 검색 결과 칩을 클릭하면 해당 종목과 해당 종목의 최신 리포트가 선택되고, 기존처럼 해당 종목의 리포트 이력과 선택 리포트 상세를 보여준다.
- 검색어를 입력하면 기존처럼 종목명과 종목코드 포함 검색 결과를 보여준다.
- 검색어를 모두 지우면 다시 `total_score` 상위 10개 추천 목록으로 돌아가며, 이미 선택된 종목의 상세는 유지한다.
- 빈 검색어로 검색 버튼을 눌러도 첫 번째 추천 종목을 자동 선택하지 않는다.

## 브라우저 시안

구현 전 브라우저 시안에서는 기존 AI 리포트 화면의 검색 패널, 결과 칩, 좌측 이력, 우측 상세 구조를 유지했다. 초기 상태는 다음처럼 배치한다.

- 상단 검색 입력은 비어 있다.
- 검색 결과 영역에는 점수 상위 10개 칩을 노출한다.
- 좌측 이력 영역에는 `선택된 종목이 없습니다.` 안내 패널을 표시한다.
- 우측 상세 영역에는 `리포트를 선택해 분석 내용을 확인하세요.` 안내 패널과 비활성 점수 카드 형태를 표시한다.

이 시안은 구현 방향 확인용이며, 실제 구현에서는 기존 CSS 클래스와 반응형 구조를 최대한 재사용한다.

## 접근 방식

권장안으로 확정한 방식은 페이지 상태만 조정하는 것이다. `src/ai-reports/model.ts`의 정규화와 대표 리포트 생성 로직은 유지한다. 변경은 `AiAnalysisReportsPage`와 해당 테스트에 한정한다.

이 방식을 선택하는 이유는 다음과 같다.

- 기존 데이터 모델과 훅의 책임을 늘리지 않는다.
- 종목별 최신 리포트 기준 점수 상위 10개는 이 페이지의 초기 검색 경험에만 필요한 표시 정책이다.
- 자동 선택 제거와 검색 입력 초기값 변경을 같은 컴포넌트 상태 흐름 안에서 검증할 수 있다.
- 기존 상세 지면, 이력 목록, 점수 카드, 에이전트 지면은 건드리지 않는다.

대안으로 모델에 추천 목록 필드를 추가할 수 있지만, 현재 요구사항은 AI 리포트 페이지의 초기 표시 정책에 가깝다. 다른 화면에서 재사용할 근거가 없으므로 이번 범위에서는 도입하지 않는다.

## 데이터 흐름

기존 훅은 그대로 `catalog.reports`와 `catalog.representativeReports`를 제공한다. 페이지는 이 데이터를 받아 다음 파생값을 만든다.

1. `representativeSearchReports`: 기존처럼 대표 리포트에 종목별 이력 수를 붙인다.
2. `hasSearchQuery`: 검색 입력을 공백 제거한 뒤 길이가 있는지 판단한다.
3. `emptySearchReports`: 검색어가 없을 때 사용할 추천 목록이다. `representativeReports`를 복사해 `totalScore` 내림차순으로 정렬하고 10개만 남긴다.
4. `visibleSearchReports`: 검색어가 있으면 기존 필터 결과를 쓰고, 검색어가 없으면 `emptySearchReports`를 쓴다.

`totalScore` 정렬 규칙은 다음과 같다.

1. `totalScore`가 있는 리포트가 없는 리포트보다 먼저 온다.
2. 둘 다 `totalScore`가 있으면 높은 값이 먼저 온다.
3. 점수가 같으면 `sortTimestamp`가 큰 최신 리포트가 먼저 온다.
4. 그래도 같으면 기존 배열 순서를 유지해 불필요한 순서 흔들림을 만들지 않는다.

## 선택 상태

현재 구현은 선택된 종목이 없거나 선택 종목이 사라졌을 때 `representatives[0]`을 자동 선택한다. 개선 후에는 자동 보정을 제거한다.

선택 상태는 다음 규칙을 따른다.

- `selectedStockCode`가 있고, 해당 종목이 현재 대표 목록에 있으면 활성 종목으로 사용한다.
- `selectedStockCode`가 없으면 활성 종목도 없다.
- `selectedStockCode`가 있었지만 새 데이터에서 사라지면 활성 종목을 `null`로 두고 미선택 안내를 보여준다.
- 검색 결과 칩의 활성 스타일은 실제 활성 종목이 있을 때만 적용한다.
- `selectedReport`는 활성 종목이 있을 때만 계산한다.

이 규칙으로 최초 진입과 데이터 재로드 모두에서 의도치 않은 상세 자동 열림을 막는다.

## 화면 구성

데이터 로드 성공 후 대표 리포트가 있으면 검색 패널은 항상 표시한다. 선택 리포트가 있을 때만 기존 상세 레이아웃을 렌더링한다.

선택 리포트가 없을 때는 기존 `state-panel` 계열 스타일을 사용해 빈 선택 상태를 보여준다.

- 좌측 영역: 종목을 선택하면 리포트 이력이 표시된다는 짧은 안내
- 우측 영역: 리포트를 선택하라는 안내와 비활성 점수 카드 4개

검색 결과 칩에는 기존처럼 종목명, 종목코드, 점수, 이력 수를 표시한다. 추천 목록임을 설명하는 긴 문구는 추가하지 않고, 검색 결과 영역 상단에 `점수 상위 추천` 수준의 짧은 보조 텍스트만 둔다.

## 오류와 빈 상태

- 로딩 상태는 기존 문구를 유지한다.
- 조회 오류 상태는 기존 오류 패널을 유지한다.
- 대표 리포트가 0개면 기존 `표시할 AI 분석 리포트가 없습니다.` 상태를 유지한다.
- 검색어가 있으나 매칭 결과가 없으면 검색 결과 칩 영역은 비어 둘 수 있다. 이때 기존 선택 상세가 있으면 유지한다.
- 검색어가 없고 대표 리포트가 10개 미만이면 가능한 개수만 보여준다.

## 테스트 계획

테스트는 구현 전에 실패하는 테스트를 먼저 추가한다.

1. `AiAnalysisReportsPage` 최초 진입 테스트
   - 검색 입력값이 빈 문자열인지 확인한다.
   - `total_score`가 높은 대표 종목 10개만 검색 결과에 표시되는지 확인한다.
   - 가장 최근 리포트가 자동 선택되지 않는지 확인한다.
   - 이력 영역과 상세 제목이 렌더링되지 않는지 확인한다.

2. 추천 칩 선택 테스트
   - 추천 칩을 클릭하면 해당 종목명이 검색 입력에 반영되는지 확인한다.
   - 해당 종목의 이력과 최신 리포트 상세가 렌더링되는지 확인한다.

3. 빈 검색 제출 테스트
   - 검색어가 빈 상태에서 검색 버튼을 눌러도 첫 추천 종목이 자동 선택되지 않는지 확인한다.

4. 기존 동작 회귀 테스트
   - 검색어 입력 후 매칭 종목을 선택하면 기존 상세, 이력, 점수 카드, 에이전트 전환이 계속 동작하는지 확인한다.

전체 검증은 다음 순서로 진행한다.

1. 대상 컴포넌트 테스트 실행
2. 전체 테스트 실행
3. 빌드 실행
4. 개발 서버에서 `/articles/finance/ai-reports` 확인 링크 제공

## 범위 제외

- Supabase 조회 컬럼과 정렬 정책은 바꾸지 않는다.
- `src/ai-reports/model.ts`의 정규화 구조는 바꾸지 않는다.
- 상세 본문, 에이전트 지면, 마크다운 렌더링 구조는 바꾸지 않는다.
- URL 쿼리로 선택 종목을 보존하는 기능은 이번 범위에 넣지 않는다.
- 추천 목록의 별도 서버 조회나 페이지네이션은 추가하지 않는다.

## 완료 기준

- 최초 진입 시 검색 입력이 비어 있다.
- 최초 진입 시 상세 본문이 자동으로 열리지 않는다.
- 검색어가 없는 상태에서 `total_score` 상위 10개 대표 종목이 검색 결과로 보인다.
- 사용자가 종목을 명시적으로 선택하면 기존 상세 경험이 정상 작동한다.
- 관련 테스트, 전체 테스트, 빌드가 통과한다.
