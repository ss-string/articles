# 컨센서스 페이지 디자인 마이그레이션 설계

## 목적

현재 컨센서스 페이지는 밝은 금융 대시보드 톤으로 구성되어 기존 포트폴리오 대시보드의 어두운 앱 셸과 시각적으로 분리되어 보인다. 이 변경은 컨센서스 기능과 데이터 구조를 유지하면서 기존 사이트의 디자인 요소로 화면을 마이그레이션한다.

## 선택 방향

A안, 기존 앱 셸 완전 흡수형을 적용한다. 컨센서스 페이지는 기존 `app-shell`, `sidebar`, `top-nav`, `dashboard`, `hero-section`, `dashboard-section`, `metric-card` 계열의 구조와 색상 체계를 사용한다.

## 화면 구조

- 좌측 사이드바와 모바일 상단 내비게이션을 복원한다.
- 브랜드 영역은 기존 `Sunghyun / Portfolio` 신호를 유지한다.
- 내비게이션 항목은 컨센서스 지면에 맞게 `Overview`, `Ranking`, `Method`, `Status`로 구성한다.
- 히어로 영역은 작은 `KRX FNGUIDE` eyebrow, `컨센서스 괴리율 랭킹` 제목, 짧은 설명, 지면 목적을 요약하는 TL;DR 패널을 포함한다.
- 지면 내 본문에는 데이터 출처나 구현 세부사항을 노출하지 않는다.
- 요약 카드는 기존 `metric-card` 시각 요소를 사용한다.
- 랭킹 테이블은 `dashboard-section` 안에 배치하고, 어두운 패널 위에서 읽히도록 행/헤더/확장 영역 색상만 조정한다.

## 보존 사항

- Supabase `krx_fnguide_consensus` 조회 방식은 유지한다.
- 괴리율 정렬, 현재가/적정주가, 1개월 대비 컨센서스 증가 표기, 확장 row, 체크포인트 가격 포함 선 그래프는 유지한다.
- 모바일에서는 기존 top-nav 아래에서 카드형 row 구조를 유지한다.

## 검증

- React 테스트로 기존 앱 셸 클래스와 컨센서스 핵심 콘텐츠가 함께 렌더링되는지 확인한다.
- 전체 테스트와 production build를 실행한다.
