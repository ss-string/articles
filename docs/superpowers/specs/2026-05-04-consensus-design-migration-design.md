# 컨센서스 페이지 디자인 마이그레이션 설계

## 목적

현재 컨센서스 지면은 포트폴리오 대시보드를 대체하지 않고, 기존 포트폴리오 페이지 안의 별도 섹션으로 접근되어야 한다. 이 변경은 기존 포트폴리오 화면을 유지하면서 내비게이션 탭으로 `컨센서스 괴리율 랭킹` 지면에 이동할 수 있게 만든다.

## 선택 방향

기존 포트폴리오 앱 셸을 유지하고, 컨센서스 지면은 `dashboard-section` 기반 섹션으로 삽입한다. 컨센서스 컴포넌트는 별도의 `app-shell`, `sidebar`, `top-nav`를 만들지 않는다.

## 화면 구조

- 좌측 사이드바와 모바일 상단 내비게이션은 기존 포트폴리오 페이지의 것을 그대로 사용한다.
- 브랜드 영역은 기존 `Sunghyun / Portfolio` 신호를 유지한다.
- 내비게이션에는 `컨센서스 괴리율 랭킹` 탭을 추가하고 `#consensus` 섹션으로 이동하게 한다.
- 컨센서스 제목은 기존 대형 히어로 제목의 약 1/3 수준으로 줄인다.
- 컨센서스 섹션에는 작은 `KRX 컨센서스 괴리율 랭킹` eyebrow, `컨센서스 괴리율 랭킹` 제목, TL;DR 문장을 포함한다.
- 컨센서스 섹션 안에는 `status-panel` 패널을 사용하지 않는다.
- 상세 확장은 별도 애로우 버튼 없이 row 선택으로 동작한다.
- 지면 내 본문에는 데이터 출처나 구현 세부사항을 노출하지 않는다.
- 요약 카드는 기존 `metric-card` 시각 요소를 사용한다.
- 랭킹 테이블은 `dashboard-section` 안에 배치하고, 어두운 패널 위에서 읽히도록 행/헤더/확장 영역 색상만 조정한다.

## 보존 사항

- Supabase `krx_fnguide_consensus` 조회 방식은 유지한다.
- 괴리율 정렬, 현재가/적정주가, 1개월 대비 컨센서스 증가 표기, 확장 row, 체크포인트 가격 포함 선 그래프는 유지한다.
- 모바일에서는 기존 top-nav 아래에서 카드형 row 구조를 유지한다.

## 검증

- React 테스트로 기존 앱 셸 클래스와 컨센서스 핵심 콘텐츠가 함께 렌더링되는지 확인한다.
- 전체 테스트와 production build를 실행한다.
