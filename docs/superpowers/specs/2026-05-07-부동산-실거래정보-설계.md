# 부동산 실거래정보 설계

## 목표

금융 외에 `부동산` 상위 카테고리를 신설하고, 하위 페이지 `실거래정보`를 추가한다. 사용자는 관심 단지-평형 목록을 카드 메뉴로 훑고, 선택한 단지의 실거래가 평균과 호가 평균 흐름을 그래프로 비교하며, 최근 실거래 중위값 이하의 현재 매물을 낮은 가격순으로 확인할 수 있어야 한다.

## 확정 방향

구현 방식은 현재 앱의 기존 패턴을 따른다.

- `api`: 5개 Supabase 테이블을 브라우저에서 직접 조회한다.
- `model`: 원천 행을 화면 모델로 정규화하고 단지-평형 기준으로 조인한다.
- `hook`: 로딩, 성공, 오류 상태를 관리한다.
- `page`: 현재 `AppShell`, 섹션 탭, `page-header`, `dashboard-section` 구조에 맞춰 화면을 렌더링한다.

Supabase view나 RPC는 이번 범위에 넣지 않는다. 현재 저장소에 DB migration 패턴이 없고, 첫 구현은 화면 요구를 검증할 수 있는 최소 구조가 더 적합하다.

## 데이터 기준

사용 테이블과 역할은 다음과 같다.

| 테이블 | 용도 |
|---|---|
| `real_estate_interest_targets` | 사용자가 보고 싶은 관심 단지-평형 목록과 정렬 순서 |
| `real_estate_complexes` | 단지명, 단지 URL, 세대수, 사용승인일 |
| `real_estate_complex_pyeong_options` | `complex_id`와 `pyeong_type` 기준 실제 표시 평형명, 전용면적 |
| `real_estate_articles` | 현재 매물. `article_number`가 고유 식별자 |
| `real_estate_pyeong_price_metrics` | 단지-평형별 실거래 기준값 시계열. 중앙값, 최저, 최고, 평균값을 포함 |

관심 단지의 화면 단위는 `complex_id + pyeong_type`이다. `real_estate_interest_targets`의 정렬 순서를 우선 적용하고, 해당 목록에 없는 단지-평형은 표시하지 않는다.

가격 그래프는 `real_estate_pyeong_price_metrics`를 시계열 기준으로 사용한다. 실거래가 평균과 호가 평균을 같은 좌표계의 두 선으로 그린다. 최신 요약 지표와 중위값 기준은 선택 단지-평형의 가장 최신 metrics 행을 사용한다.

## 중위값 이하 매물 기준

`중위값 이하 매물`은 선택 단지-평형의 최신 `real_estate_pyeong_price_metrics` 행에 있는 최근 실거래 중위값을 기준으로 한다.

```text
현재 매물 호가 <= 최신 실거래 중위값
```

목록에는 `real_estate_articles`의 현재 매물 중 위 조건을 만족하는 매물만 노출한다. 정렬은 낮은 호가순이다. 카드에는 매물 가격, 단지/동/층 등 사용 가능한 위치 정보, 매물번호, 중위값 대비 차이를 표시한다.

## 화면 구조

부동산 카테고리는 기존 네비게이션 구조를 확장한다.

- 상위 메뉴: `메인`, `금융`, `부동산`
- 부동산 기본 경로: `/real-estate/transactions`
- 부동산 하위 탭: `실거래정보`
- 페이지 헤더: 킥커 `부동산 가격 모니터링`, 제목 `실거래정보`

본문은 승인된 B안 구조를 현재 앱 스타일에 맞춰 사용한다.

1. 왼쪽 관심단지 카드 메뉴
   - 단지명, 표시 평형명, 세대수, 사용승인일, 중위값 이하 매물 수, 현재 호가 평균을 표시한다.
   - 선택된 카드는 기존 활성 메뉴처럼 녹색 계열 배경과 테두리로 구분한다.
2. 오른쪽 선택 단지 요약
   - 최신 실거래 평균, 현재 호가 평균, 실거래 중위값, 현재 매물 수를 요약 카드로 표시한다.
3. 가격 비교 그래프
   - `real_estate_pyeong_price_metrics`의 실거래가 평균과 호가 평균을 선 그래프로 표시한다.
   - 그래프는 외부 차트 라이브러리 없이 SVG로 구현한다.
4. 중위값 이하 매물 목록
   - 최신 실거래 중위값 이하 매물만 낮은 호가순으로 표시한다.
   - 매물이 없으면 선택 단지 기준에 맞는 매물이 없다는 빈 상태를 보여준다.

모바일에서는 관심단지 카드 메뉴를 상단 가로 스크롤 목록으로 바꾸고, 요약 카드와 그래프, 매물 목록을 단일 컬럼으로 쌓는다.

## 상태 처리

로딩 상태에서는 기존 `state-panel` 톤으로 데이터를 불러오는 중임을 표시한다. Supabase 환경변수가 없거나 조회에 실패하면 오류 메시지를 표시한다. 관심단지 목록이 비어 있으면 페이지 본문에 표시할 관심 단지가 없다는 빈 상태를 표시한다. 선택된 관심 단지의 metrics 또는 매물이 부족한 경우에도 페이지 전체를 실패시키지 않고 해당 영역만 빈 상태로 둔다.

## 변경 범위

- `src/navigation.ts`: `real-estate` 섹션 타입, 라우트, 표시 네비게이션, 기본 섹션 경로를 추가한다.
- `src/components/AppShell.tsx`: 금융 전용 탭을 섹션별 탭 렌더링으로 일반화한다.
- `src/App.tsx`: `/real-estate/transactions` 경로에서 새 페이지를 렌더링한다.
- `src/real-estate/api.ts`: 5개 Supabase 테이블 조회 함수를 추가한다.
- `src/real-estate/model.ts`: 관심 단지, 가격 metrics, 현재 매물 행을 화면 모델로 정규화한다.
- `src/real-estate/useRealEstateTransactions.ts`: 조회 상태와 정규화된 페이지 데이터를 제공한다.
- `src/components/RealEstateTransactionsPage.tsx`: 승인된 B안 화면을 구현한다.
- `src/styles.css`: 부동산 카드 메뉴, 그래프, 매물 목록, 반응형 스타일을 추가한다.

## 테스트 기준

테스트는 기존 Vitest와 Testing Library 패턴을 따른다.

- 네비게이션 테스트: `/real-estate/transactions`가 정상 라우트로 인식되고, 부동산 상위 메뉴와 실거래정보 탭이 활성화되는지 검증한다.
- 모델 테스트: 5개 테이블 행을 `complex_id + pyeong_type` 기준으로 조인하고, 최신 metrics 행의 중위값 이하 매물만 낮은 가격순으로 남기는지 검증한다.
- hook 테스트: Supabase 환경변수 오류, 조회 성공, 조회 실패 상태를 검증한다.
- 페이지 테스트: 관심단지 카드 선택 시 요약 지표, 그래프 제목, 중위값 이하 매물 목록이 선택 단지 기준으로 바뀌는지 검증한다.

## 제외 범위

- DB migration, Supabase view, RPC 생성
- 관심단지 추가/삭제/정렬 편집 UI
- 매물 크롤링 또는 metrics 생성 로직
- 상세 팝업, 지도, 외부 사이트 iframe
- 금융 페이지의 기존 데이터 모델 변경
