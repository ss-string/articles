# 부동산 90일 실거래 그래프와 활성 매물 범위 설계

## 목표

부동산 실거래정보 화면을 새 DB 구조에 맞춰 갱신한다. 사용자는 단지 단위로 관심 대상을 선택하고, 최근 90일 실거래 이력과 현재 활성 매물 가격 범위를 한 화면에서 비교할 수 있어야 한다. 이전 수집에는 있었지만 이번 수집에서 사라진 매물은 현재 매물 목록, 매물 개수, 매물 가격 범위, 그래프 보조 정보에 포함하지 않는다.

## 확정 방향

화면의 좌측 목록은 `complex_id` 기준으로 묶는다. 같은 단지에 여러 `pyeong_type`이 관심 대상으로 들어오면 좌측 카드 하나로 표시하고, 카드 안의 평형명은 `84C / 85A / 85B`처럼 합쳐 보여준다.

그래프 데이터는 DB의 `real_estate_pyeong_price_metrics.chart_data`를 사용한다. 조회 기준은 다음과 같이 고정한다.

```text
complex_id = 선택 단지의 complex_id
pyeong_type = 관심 대상 pyeong_type
trade_type = 'A1'
window_months = 3
```

같은 단지에 여러 평형이 있으면 클라이언트에서 각 평형의 `chart_data`를 병합한다. 서버 view나 RPC는 이번 범위에 넣지 않는다.

## 데이터 규칙

### 활성 매물 필터

매물 목록과 활성 매물 집계는 반드시 `real_estate_articles.is_active = true`만 사용한다.

다음 매물은 제외한다.

- `is_active=false`
- `removed_at` 값이 있는 매물
- 선택 단지의 관심 평형에 속하지 않는 매물
- 매매가 아닌 매물

API 조회에서도 `is_active=eq.true`를 적용하고, 모델 정규화 단계에서도 `is_active=false` 또는 `removed_at` 값이 있으면 다시 제외한다.

### chart_data 계약

`real_estate_pyeong_price_metrics.chart_data`는 다음 계약을 따른다.

```text
chart_data.series: 실거래가 점/라인
chart_data.markers.latestRealPrice: 최신 실거래가 마커
chart_data.markers.highestRealPrice: 최고 실거래가 마커
chart_data.activeListingRange: 현재 활성 매물 가격 범위
```

기존 `actual_average_price`, `asking_average_price`, `actual_median_price` 기반 그래프는 이번 화면의 주 데이터가 아니다. 필요하면 호환용 fallback으로만 사용한다.

### 단지 단위 병합

같은 `complex_id`에 여러 `pyeong_type`이 있으면 단지 카드 하나로 병합한다.

병합 규칙은 다음과 같다.

- `series`: 각 평형의 `chart_data.series`를 모은 뒤 `tradeDate + dealPrice` 기준으로 중복 제거하고 날짜순으로 정렬한다.
- `latestRealPrice`: 병합된 `series` 중 가장 최신 `tradeDate`의 실거래로 다시 계산한다. 같은 날짜가 여러 건이면 더 높은 가격을 우선한다.
- `highestRealPrice`: 병합된 `series` 중 가장 높은 `dealPrice`로 다시 계산한다.
- `activeListingRange.minDealPrice`: 각 평형의 활성 매물 범위와 활성 매물 목록을 합친 값 중 최저가를 사용한다.
- `activeListingRange.maxDealPrice`: 각 평형의 활성 매물 범위와 활성 매물 목록을 합친 값 중 최고가를 사용한다.
- `activeListingRange.count`: 각 평형의 활성 매물 건수를 합산한다.
- `currentArticles`: 각 평형의 활성 매물을 합쳐 낮은 가격순으로 정렬하고, 행에는 해당 평형명을 함께 표시한다.

병합 후에도 y축은 실거래가와 활성 매물 범위를 함께 포함해야 한다.

```text
domain source =
  chart_data.series[*].dealPrice
  activeListingRange.minDealPrice
  activeListingRange.maxDealPrice
```

## 화면 구조

기존 부동산 페이지의 2열 구조를 유지한다.

1. 좌측 단지 목록
   - `complex_id` 기준 단지 카드 하나를 표시한다.
   - 단지명, 병합 평형명, 세대수, 사용승인일, 활성 매물 건수, 활성 매물 가격 범위를 표시한다.
   - 카드 높이는 내용 높이만 차지한다. 오른쪽 그래프 영역 높이에 맞춰 늘어나면 안 된다.
2. 우측 요약 카드
   - 최신 실거래
   - 최고 실거래
   - 현재 활성 매물 건수
3. 우측 그래프 패널
   - 헤더는 `최근 90일 · 매매` 단일 기준만 표시한다.
   - 1년/3년/5년/전체 기간 탭은 표시하지 않는다.
   - 실거래는 점과 선으로 표시한다.
   - 최신 실거래와 최고 실거래는 별도 마커로 표시한다.
   - 활성 매물 가격 범위는 그래프 우측 끝, 최신 기준선 오른쪽에 작은 초록색 범위 박스로 표시한다.
   - 범위 박스 위에는 `활성 매물 가격범위` 라벨을 표시하고, 다음 줄에 `15.2억-16.5억` 같은 범위를 표시한다.
   - 범위 박스 라벨이 그래프나 박스와 겹치지 않도록 플롯 상단 여백을 확보한다.
4. 활성 매물 목록
   - `is_active=true`이고 `removed_at`이 없는 매물만 표시한다.
   - 병합 단지에서는 각 행에 평형명, 동, 층, 가격, 매물번호를 표시한다.

## 그래프 렌더링

x축은 실거래 건수 기준 등간격이 아니라 날짜 기준으로 계산한다. 기준 기간은 `updated_at` 또는 최신 metrics 기준일에서 최근 90일이다. 실거래가 1건 또는 2건뿐이어도 전체 폭을 억지로 가로지르는 선처럼 보이면 안 된다.

활성 매물 가격 범위 박스는 그래프 전체 폭을 덮는 레터박스가 아니다. 최신 시점 기준선 오른쪽에 작은 박스로 표시한다. 박스의 y 위치와 높이는 `activeListingRange.minDealPrice`, `activeListingRange.maxDealPrice`를 y축 domain에 투영해 계산한다. 가격 범위가 매우 좁아 박스가 보이지 않는 경우에는 최소 시각 높이만 둔다.

## 가격 표기

화면 표기는 억 단위를 우선한다.

```text
1250000000 -> 12.5억
1520000000 -> 15.2억
1650000000 -> 16.5억
```

tooltip과 마커 라벨은 원 단위 원본값보다 억 단위 가격, 날짜, 구분을 우선 표시한다.

## 빈 상태

빈 상태는 그래프 패널 높이를 유지한 채 영역별로 표현한다.

- `series.length === 0`: `최근 90일 실거래 없음`
- `activeListingRange.count === 0`: `현재 활성 매물 없음`
- 둘 다 없으면 그래프 영역은 유지하고 빈 상태 문구만 표시한다.

활성 매물 범위가 없으면 우측 초록 범위 박스를 표시하지 않는다.

## 검수 기준

- 이전 수집에는 있었지만 이번 수집에서 사라진 매물이 UI에 보이지 않는다.
- 매물 개수와 매물 가격 범위는 `is_active=true` 매물만 기준으로 계산된다.
- 래미안크레시티처럼 여러 관심 평형이 있는 단지는 좌측 목록에서 하나로 묶인다.
- 병합 단지는 여러 평형의 `chart_data.series`가 클라이언트에서 병합되어 그래프에 표시된다.
- 그래프에는 최근 90일 실거래가 날짜 기준 위치로 표시된다.
- 최신 실거래와 최고 실거래 마커가 표시된다.
- 활성 매물 가격 범위가 그래프 우측 작은 초록 박스로 표시되고, `활성 매물 가격범위` 라벨과 가격 범위가 겹치지 않는다.
- 1년/3년/5년/전체 탭이 표시되지 않는다.
