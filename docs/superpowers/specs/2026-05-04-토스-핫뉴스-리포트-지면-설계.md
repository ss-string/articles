# 토스 핫뉴스 리포트 지면 설계

## 배경

`toss_wts_hot_news_reports` 테이블의 공개 조회 권한을 열어 확인한 결과, 테이블에는 핫뉴스 리포트를 구성하기에 충분한 정규화 데이터가 들어 있다. 지면은 뉴스 목록을 간단한 카드로 보여주고, 사용자가 카드를 선택하면 해당 row의 정보를 리포트 팝업으로 재구성해 보여준다.

확인된 주요 컬럼은 다음과 같다.

- `id`
- `issue_date`
- `title`
- `perspective`
- `tldr`
- `key_articles`
- `company_news_evidence`
- `interpretation`
- `report_payload`
- `created_at`
- `updated_at`

`report_payload`에는 `title`, `tldr`, `keyArticles`, `interpretation`, `companyNewsEvidence`가 이미 정규화되어 있으므로 이를 우선 사용한다. 값이 없거나 형식이 맞지 않는 경우에는 top-level 컬럼을 대체값으로 사용한다.

## 목표

- `toss_wts_hot_news_reports`를 Supabase에서 조회해 핫뉴스 리포트 카드 지면을 추가한다.
- 각 카드는 제목, 날짜, 관점, 요약을 간결하게 보여준다.
- 카드를 선택하면 row 기반 리포트 팝업을 띄운다.
- 팝업은 기존 앱의 어두운 대시보드 톤, 8px 라운드, 청록색 강조 색상을 유지한다.
- 팝업 뒤 화면에는 딤드 오버레이를 적용한다.
- 주요 기사에 링크가 있으면 외부 링크로 연결한다.

## 범위

이번 작업의 범위는 새 핫뉴스 리포트 지면과 팝업을 앱 안에 추가하는 것이다. 기존 컨센서스 랭킹 지면의 계산 방식이나 UI는 변경하지 않는다.

## 구조

새 기능은 기존 컨센서스 기능과 같은 경계로 분리한다.

- `src/hot-news/api.ts`: Supabase `toss_wts_hot_news_reports` 조회
- `src/hot-news/model.ts`: raw row를 표시 모델로 정규화
- `src/hot-news/useHotNewsReports.ts`: 조회 상태 관리
- `src/components/HotNewsReportsPage.tsx`: 카드 지면 렌더링
- `src/components/HotNewsReportModal.tsx`: 딤드와 리포트 팝업 렌더링
- `src/App.tsx`: 지면 연결
- `src/content.ts`: 내비게이션 항목 추가

Supabase 설정은 기존 컨센서스 기능의 환경변수 처리 방식을 따른다.

## 표시 모델

정규화된 표시 모델은 다음 구조를 사용한다.

```ts
type HotNewsReport = {
  id: string;
  issueDate: string | null;
  displayDate: string;
  title: string;
  perspective: string | null;
  tldr: string[];
  interpretation: string | null;
  keyArticles: HotNewsArticle[];
  companyEvidence: HotNewsCompanyEvidence[];
};

type HotNewsArticle = {
  title: string;
  link: string | null;
};

type HotNewsCompanyEvidence = {
  company: string;
  code: string | null;
  position: 'bull' | 'bear' | 'neutral' | string | null;
  evidence: string[];
  links: string[];
};
```

## 데이터 우선순위

각 항목은 `report_payload`를 우선 사용하고 top-level 컬럼을 대체값으로 사용한다.

- 제목: `report_payload.title` → `title`
- 요약: `report_payload.tldr` → `tldr`
- 주요 기사: `report_payload.keyArticles` → `key_articles`
- 기업별 근거: `report_payload.companyNewsEvidence` → `company_news_evidence`
- 시장 해석: `report_payload.interpretation` → `interpretation`
- 관점: `perspective`
- 날짜: `issue_date`

## 카드 지면

카드에는 반복 조회에 필요한 핵심 정보만 표시한다.

- 제목: `title`
- 날짜: `issue_date`를 한국어 날짜 형식으로 표시
- 관점: `perspective`
- 요약: `tldr[0]`

카드는 버튼처럼 동작한다. 마우스 클릭과 키보드 `Enter`, `Space` 입력으로 팝업을 연다.

## 리포트 팝업

팝업은 다음 섹션으로 구성한다.

- `TL;DR`: `tldr` 배열을 목록으로 표시
- `시장 해석`: `interpretation` 문단 표시
- `기업별 근거`: 회사명, 종목코드, 포지션, 근거 목록, 관련 링크 표시
- `주요 기사`: 기사 제목 목록 표시

주요 기사에 `link`가 있으면 제목 전체를 클릭 가능한 외부 링크로 표시한다. 새 탭으로 열고 `target="_blank"`, `rel="noreferrer"`를 적용한다. 링크가 없으면 일반 텍스트로 표시한다.

## 팝업 인터랙션

- 팝업이 열리면 뒤 화면에 딤드 오버레이를 적용한다.
- 닫기 버튼은 기존 앱의 테두리 버튼 톤과 8px 라운드를 사용한다.
- 닫기 방식은 X 버튼, 딤드 클릭, `Esc`를 지원한다.
- 팝업이 열려 있는 동안 body 스크롤을 잠근다.
- 팝업에는 `role="dialog"`와 `aria-modal="true"`를 적용하고 제목과 연결한다.

## 오류와 빈 상태

- Supabase 환경변수가 없으면 기존과 같은 방식으로 오류 메시지를 표시한다.
- 조회 실패 시 `state-panel error` 스타일을 사용한다.
- 표시 가능한 row가 없으면 “표시할 핫뉴스 리포트가 없습니다.”를 표시한다.
- 제목이 없는 row는 제외한다.
- 배열이어야 하는 값이 배열이 아니면 빈 배열로 처리한다.
- 링크 문자열이 비어 있으면 링크 없는 항목으로 처리한다.
- 날짜 파싱이 실패하면 원문 문자열을 표시한다.
