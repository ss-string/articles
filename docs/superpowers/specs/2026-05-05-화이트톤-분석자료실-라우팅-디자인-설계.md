# 화이트톤 분석자료실 라우팅 디자인 설계

## 목적

현재 단일 포트폴리오 대시보드형 화면을 화이트톤의 블로그/분석 자료실 디자인으로 마이그레이션한다. 사용자는 좌측 사이드바와 상단 탭을 통해 `메인`과 `금융` 분석 화면을 분리해서 탐색한다. 화면은 개인명을 드러내지 않고 `분석자료실`이라는 개인 서재형 브랜드 톤을 사용한다.

## 최종 방향

디자인 방향은 에디토리얼 인덱스형이다. 좌측에는 카테고리 사이드바를 두고, 본문은 흰 배경과 얇은 보더, 절제된 청록색 활성 상태를 사용한다. 데스크탑과 모바일 모두 동일한 정보 구조를 유지한다.

모바일에서는 사이드바가 좌측 드로어로 열리고, 닫기 버튼은 두지 않는다. 사용자는 우측 딤드 영역을 선택해 사이드바를 닫는다.

## 브랜드와 전체 톤

- 사이드바 브랜드는 `분석자료실 / Research Library`를 사용한다.
- `Sunghyun` 등 개인명은 화면에 노출하지 않는다.
- 기본 배경은 밝은 오프화이트 계열을 사용한다.
- 본문 카드와 패널은 흰색 배경, 얇은 베이지/그레이 보더, 8px 이하의 보더 반경을 사용한다.
- 활성 상태는 청록색 텍스트와 옅은 청록색 배경으로 표현한다.
- 상위 카테고리가 열린 상태는 화살표 없이 배경색으로만 표현한다.

## 내비게이션 구조

최종 정보 구조는 다음과 같다.

```txt
분석자료실

메인
  매크로 레짐

금융
  핫 뉴스 분석 리포트
  컨센서스 괴리율 랭킹
  AI 분석 리포트
```

`메인`을 선택한 경우에만 `매크로 레짐` 하위 메뉴를 표시한다. `금융`을 선택한 경우에만 금융 하위 3개 메뉴를 표시한다. 한 상위 카테고리가 활성화된 동안 다른 상위 카테고리의 하위 메뉴는 숨긴다.

## 라우팅

라우팅은 섹션 해시 이동이 아니라 별도 페이지 이동을 기준으로 한다.

```txt
/ 또는 /main/macro-regime
  메인 > 매크로 레짐

/finance/hot-news
  금융 > 핫 뉴스 분석 리포트

/finance/consensus
  금융 > 컨센서스 괴리율 랭킹

/finance/ai-reports
  금융 > AI 분석 리포트
```

기본 진입은 `메인 > 매크로 레짐`이다. `/`로 접근하면 매크로 레짐 화면을 렌더링하거나 해당 경로로 이동시킨다. 본문에는 `/finance/hot-news`, `/main/macro-regime` 같은 경로 문자열을 노출하지 않는다. 대신 `시장 국면 분석`, `금융 이슈`, `금융 밸류에이션`, `자동 분석 요약` 같은 설명형 라벨을 사용한다.

## 최상단 행

본문 영역 최상단에는 공통 행을 추가한다.

- 좌측에는 사이드바 열기/닫기 버튼만 배치한다.
- `분석자료실` 텍스트는 최상단 행에 표시하지 않는다.
- 우측에는 추후 로그인, 설정 등 전역 액션을 배치할 수 있는 영역을 둔다.
- 데스크탑과 모바일 모두 같은 역할의 최상단 행을 사용한다.
- 사이드바 토글 버튼은 본문 제목 영역에 두지 않는다.

## 데스크탑 사이드바

데스크탑은 좌측 고정 사이드바를 사용한다. 최상단 행의 좌측 버튼으로 사이드바를 열고 닫는다.

- 사이드바가 열린 상태에서는 좌측 내비게이션과 본문이 함께 보인다.
- 사이드바가 닫힌 상태에서는 본문이 전체 폭을 사용한다.
- 토글 상태는 사용자가 반복 탐색할 수 있도록 유지하는 방향이 적합하다.
- 상위 카테고리 옆에는 펼침 화살표를 두지 않는다.

## 모바일 사이드바

모바일에서도 데스크탑과 동일한 사이드바 정보 구조를 사용한다. 다만 화면 폭 때문에 좌측 드로어로 표시한다.

- 최상단 행의 좌측 버튼을 누르면 사이드바가 좌측에서 열린다.
- 사이드바 내부에는 별도 `X` 닫기 버튼을 두지 않는다.
- 사이드바 오른쪽의 딤드 영역을 선택하면 닫힌다.
- 사이드바 메뉴를 선택하면 해당 화면으로 이동하고 드로어는 닫힌다.
- 본문 상단의 금융 언더라인 탭은 모바일에서도 유지한다.

## 금융 상단 언더라인 탭

금융 하위 페이지에는 본문 제목 아래에 상단 언더라인 탭을 둔다.

```txt
핫 뉴스 분석 리포트 | 컨센서스 괴리율 랭킹 | AI 분석 리포트
```

- 현재 탭은 청록색 텍스트와 하단 라인으로 표시한다.
- 비활성 탭은 중립 텍스트 색상과 투명 하단 라인을 사용한다.
- 데스크탑에서는 전체 명칭을 표시한다.
- 모바일에서는 가로 스크롤 가능한 언더라인 탭으로 유지한다.
- 모바일 탭 라벨은 화면 폭을 고려해 축약할 수 있으나, 접근성 라벨에는 전체 명칭을 유지한다.
- 금융 하위 페이지가 아닌 `메인 > 매크로 레짐`에서는 이 탭을 표시하지 않는다.

## 심볼 규칙

메뉴 앞에는 의미 중심 심볼을 사용한다. 확정 심볼은 다음과 같다.

```txt
⌂ 메인
◷ 매크로 레짐
◆ 금융
≡ 핫 뉴스 분석 리포트
↗ 컨센서스 괴리율 랭킹
⌁ AI 분석 리포트
```

심볼은 각 메뉴의 왼쪽에 작은 배지 형태로 배치한다. 활성 메뉴의 심볼 배지는 옅은 청록색 배경과 청록색 텍스트를 사용한다. 비활성 심볼은 중립 배경과 중립 텍스트를 사용한다.

## 화면별 본문

### 메인 > 매크로 레짐

기본 진입 화면이다. 시장 국면 분석 라벨과 `매크로 레짐` 제목을 사용한다. 기존 매크로 레짐 카드, 상세 팝업, 핵심 지표 정보 구조는 유지하되 밝은 톤의 카드와 패널로 재스킨한다.

### 금융 > 핫 뉴스 분석 리포트

기존 핫뉴스 리포트 기능을 금융 하위 페이지로 이동한다. 본문 제목은 `핫 뉴스 분석 리포트`를 사용한다. 카드 목록과 상세 팝업은 유지하되, 금융 상단 언더라인 탭을 통해 다른 금융 화면으로 이동할 수 있게 한다.

### 금융 > 컨센서스 괴리율 랭킹

기존 컨센서스 괴리율 랭킹 기능을 금융 하위 페이지로 이동한다. 본문 제목은 `컨센서스 괴리율 랭킹`을 사용한다. 요약 카드, 랭킹 테이블, 상세 팝업은 유지하되 밝은 테이블 스타일로 조정한다.

### 금융 > AI 분석 리포트

신규 금융 하위 페이지다. 초기 구현에서는 라우팅, 내비게이션, 빈 상태 또는 준비 중 상태를 제공한다. 실제 AI 분석 리포트 데이터 모델과 조회 방식은 별도 구현 범위로 다룬다.

## 오류와 빈 상태

- 각 페이지는 로딩, 오류, 빈 상태를 밝은 톤 패널로 표시한다.
- 오류 상태는 과한 붉은 배경 대신 옅은 경고 배경과 명확한 문장으로 표현한다.
- AI 분석 리포트는 데이터가 없을 때 준비 중 또는 표시할 리포트 없음 상태를 노출한다.

## 접근성

- 사이드바 토글 버튼에는 명확한 `aria-label`을 제공한다.
- 모바일 드로어가 열리면 딤드 영역 선택으로 닫을 수 있어야 한다.
- 금융 언더라인 탭은 현재 페이지를 보조 기술이 알 수 있도록 현재 상태를 표현한다.
- 모바일에서 축약 탭 라벨을 사용하더라도 전체 명칭은 접근성 라벨로 유지한다.
- 키보드 포커스는 토글 버튼, 사이드바 메뉴, 상단 탭, 모달 닫기 동작을 자연스럽게 순환해야 한다.

## 검증

- 기본 진입 시 매크로 레짐 화면이 렌더링되는지 확인한다.
- `메인` 활성 상태에서는 `매크로 레짐` 하위 메뉴만 표시되는지 확인한다.
- `금융` 활성 상태에서는 금융 하위 3개 메뉴만 표시되는지 확인한다.
- 데스크탑 사이드바 토글 시 본문 폭이 확장되고 복구되는지 확인한다.
- 모바일 드로어가 최상단 행 버튼으로 열리고, 우측 딤드 영역 선택으로 닫히는지 확인한다.
- 금융 하위 페이지에서 언더라인 탭 전환이 라우팅과 본문 제목을 함께 변경하는지 확인한다.
- 전체 테스트와 프로덕션 빌드를 실행한다.
